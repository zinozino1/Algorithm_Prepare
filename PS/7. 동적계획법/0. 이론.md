1. 동적계획법

하나의 문제를 단 한 번만 풀도록 하는 알고리즘 -> 기존에 풀었던 문제는 저장해놓고 동일한 문제가 나오면 꺼내 쓴다.

* DP는 다음의 가정 하에 사용할 수 있다.
1) 큰 문제를 작은 문제로 나눌 수 있다.
2) 작은 문제에서 구한 정답은 그것을 포함하는 큰 문제에서도 동일하다.

즉 쉽게 말하면 크고 어려운 문제가 있으면 그것을 잘게 나누어서 해결한 뒤에 처리하여 나중에 전체 답을 구하는 것. 이는 분할정복과 유사하나, '메모이제이션'이 사용된다는 점에서 분할 정복과 다르다.

* bottom-up
-> 작은 문제에 대한 답을 큰 문제의 답의 일부로 사용하는 것

* top-down
-> 재귀와 메모이제이션 활용

** 공통점 : 메모이제이션을 활용한다는 것 & 문제에 대한 점화식을 정의해놓고 코드를 작성해야한다는 것

**** 다이나믹 문제를 푸는데 탑다운식의 방법도 있다 정도. DP 문제는 거의 대부분은 바텀업방식으로 해결하는게 정석이다


*************
1 - 이 문제를 DP로 풀 것인가?
-> 우선 기본 이터레이션과 완전탐색을 고려해보자. 경우의 수가 너무 많거나 재귀의 깊이가 너무 깊을 때 그리디와 디피를 고려.
  1-1 그리디? -> 파악하기 굉장히 어렵다. 일단 순간의 선택이 최선이 되는 경우를 고려한 후 반례를 찾아야함
=> 그리디와 완전탐색으로 푸는 것이 어렵다고 판단되면 DP로 푸는것이 이상적


2 - DP 배열엔 무엇이 들어가야하는가?
-> 아주 작게 쪼개는 과정
-> 거의 100% 문제에서 요구하는 바가 들어가야함(7m짜리 랜선을 잘랐을 때 경우의수(경우의 수 들어가야함) or ~를 마지막으로하는 증가수열의 최대 길이(길이값 들어가야함)) + 일단 아직까지는 ~를 마지막으로할 때 ex) 마지막 선이 1m, 2m일 때 or 마지막으로 끝나는 증가수열이 7일 때 등등
** dp[0] -> 원소 하나일 때 ~값 , dp[1] -> 원소 두개일 때 ~값 이런식으로 생각하면 된다.


3 - DP의 규칙성(점화식)은 무엇인가?
-> 사실상 핵심. 일단 dp를 직접 계산해서 나열해보고 문제의 규칙에 따라 점화식을 파악해보자



2. TIP

1) 점화식을 세우는 것이 가장 중요하다

2) TOP - DOWN VS BOTTOM - UP

TOP - DOWN 방식 에 메모이제이션을 했다는 가정하에 시간복잡도는 같다.
하지만 실제 걸리는 시간은 TOP - DOWN DP가 더 길다고 일반적으로 알려져 있다.
재귀 DP의 장점은 점화식 그대로 호출이 되기 때문에 형식/순서에 얽매이지 않는다.
for문 DP의 장점은 시간이 (조금은) 적게 걸린다는 것이다. 그렇기 때문에 문제에 따라서 둘 중 어떻게 할지 잘 정해야 한다.
둘다 할 줄 알아야 된다.
